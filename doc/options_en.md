# Neviweb130 Integration Options

This page describes all available configuration options for the **Neviweb130** integration.

---

## ðŸ”§ Available parameters

### **Account prefix**
Unique identifier used to distinguish multiple Neviweb accounts in Home Assistant.

### **Log level**
Controls the amount of information written to the logs:
- debug
- info
- warning
- error
- critical

### **Maximum log size**
Maximum size of the log file before rotation.

### **Backup count**
Number of rotated log files to keep.

### **Reset log on startup**
Clears the log file when the integration starts.

### **Download diagnostics**
Downloads a ZIP file containing logs and diagnostic information.

### **Reload integration**
Reloads the integration without restarting Home Assistant.

### **Migrate numeric unique_id**
Converts old numeric identifiers to stable text-based identifiers.

### **Backup mode**
- **full**: full backup
- **partial**: partial backup

### **Backup root folder**
Path of the folder to back up.
